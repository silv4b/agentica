# Dart

## Comandos

- Executar: `dart run`
- Testar: `dart test`
- Linter: `dart analyze`
- Formatar: `dart format`

## Estilo de Código

- Siga o guia de estilo Dart (effective dart)
- Use construtores `const` quando possível
- Prefira `final` em vez de `var`
- Use tipos anuláveis com null safety adequado

## Melhores Práticas

- Use `async`/`await` para código assíncrono
- Prefira streams Dart para código reativo
- Use classes `sealed` para hierarquias fechadas
- Use pattern matching introduzido no Dart 3
