# Angular

## Comandos

- Executar: `ng serve`
- Testar: `ng test`
- Linter: `ng lint`
- Build: `ng build`

## Estilo de Código

- Siga o guia de estilo oficial do Angular
- Use componentes standalone (NgModules são legados)
- Use signals para gerenciamento de estado
- Prefira formulários reativos em vez de template-driven
- Use formulários reativos tipados

## Melhores Práticas

- Use lazy loading para módulos/rotas de funcionalidades
- Implemente a estratégia OnPush de change detection
- Use o pipe `async` em vez de subscriptions manuais
- Estrutura de pastas por funcionalidade (folder-by-feature)
- Use Angular CLI para geração de código
