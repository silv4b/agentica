# NestJS

## Commands

- Run: `npm run start:dev`
- Test: `npm run test`
- Lint: `npm run lint`
- Build: `npm run build`

## Code Style

- Follow the official NestJS style guide
- Use modules for feature organization
- Prefer classes with decorators over plain objects
- Use DTOs and validation pipes

## Best Practices

- Use dependency injection as the primary pattern
- Organize code in modules (feature-based)
- Use guards for authentication/authorization
- Use interceptors for logging and transformation
- Use pipes for input validation
- Use TypeORM or Prisma for database access
