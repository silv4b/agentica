# Flutter

## Commands

- Run: `flutter run`
- Test: `flutter test`
- Analyze: `flutter analyze`
- Build: `flutter build [platform]`

## Code Style

- Use stateless widgets by default
- Extract reusable widgets into separate files
- Use provider, riverpod, or bloc for state management
- Follow material design guidelines
- Use `const` constructors where possible

## Best Practices

- Use `MediaQuery` and `LayoutBuilder` for responsive layouts
- Handle loading, error, and empty states in all async views
- Use `GoRouter` or `Navigator 2.0` for routing
- Use `flutter_lints` or custom lint rules
- Write widget and unit tests for critical paths
