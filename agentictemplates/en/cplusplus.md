# C++

## Commands

- Build: `cmake --build .` or `make`
- Test: `ctest`

## Code Style

- Follow the C++ Core Guidelines
- Use RAII for resource management
- Prefer smart pointers over raw pointers
- Use `constexpr` where applicable
- Use C++20/23 features when possible

## Best Practices

- Use CMake for building
- Separate header files from implementation
- Use namespaces to avoid name collisions
- Leverage STL containers and algorithms
- Use modern C++ over C-style patterns
