# Dart

## Commands

- Run: `dart run`
- Test: `dart test`
- Lint: `dart analyze`
- Format: `dart format`

## Code Style

- Follow the Dart style guide (effective dart)
- Use `const` constructors where possible
- Prefer `final` over `var`
- Use nullable types with proper null safety

## Best Practices

- Use `async`/`await` for asynchronous code
- Prefer Dart streams for reactive code
- Use `sealed` classes for sealed hierarchies
- Use pattern matching introduced in Dart 3
